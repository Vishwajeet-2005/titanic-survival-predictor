🧠 How Random Forest Works

A Random Forest builds multiple decision trees, each trained on a random subset of data and features. For prediction, each tree "votes" on an outcome, and the majority vote wins. This reduces overfitting and improves generalization vs. a single decision tree.

🔌 Flask API Documentation

The Titanic predictor comes with a Flask backend API. Start the server with python app.py and interact with the endpoints below. All responses are JSON.

POST /api/predict
// Request Body (JSON) { "pclass": 1, // 1, 2, or 3 "sex": "female", // "male" or "female" "age": 28, // float "fare": 52.0, // float (British pounds) "sibsp": 0, // int: siblings/spouses aboard "parch": 0, // int: parents/children aboard "embarked": "S" // "S", "C", or "Q" } // Response { "success": true, "prediction": { "survived": true, "survival_probability": 0.8241, "confidence": 0.6482 }, "model_info": { "type": "RandomForest", "n_trees": 80, "accuracy": 0.7989 } }
GET /api/stats
// Returns dataset statistics for charts and EDA { "success": true, "stats": { "survival_rate": 0.3838, "by_sex": { "female": 0.742, "male": 0.188 }, "by_class": { "1": 0.629, "2": 0.472, "3": 0.242 }, "total": 891 } }
POST /api/batch-predict
// Predict multiple passengers at once { "passengers": [ { "pclass": 1, "sex": "female", "age": 25, "fare": 80, ... }, { "pclass": 3, "sex": "male", "age": 22, "fare": 7.25, ... } ] }
GET /api/model-info
// Returns model metadata and feature importances { "model": { "type": "RandomForest", "n_trees": 80, "features": ["pclass", "sex_encoded", ...], "accuracy": 0.7989, "feature_importance": { "sex_encoded": 0.28, ... } } }
